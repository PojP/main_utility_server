# RSS Reader — HTTP API

Base URL: `http://<host>:8080`

CORS открыт для всех origin (`*`). Все ответы — JSON, кодировка UTF-8.

## GET /api/news

Список статей с cursor-пагинацией.

### Query-параметры

| Параметр | Тип    | По умолчанию | Описание |
|----------|--------|-------------|----------|
| `limit`  | int    | 20          | Количество статей (1–100) |
| `after`  | int64  | —           | Вернуть статьи **старше** указанного ID (прокрутка вниз) |
| `since`  | int64  | —           | Вернуть статьи **новее** указанного ID (обновление ленты) |
| `tags`   | string | —           | Фильтр по тегам через запятую, семантика OR |

`after` и `since` взаимоисключающие. Если указаны оба — используется `after`.

### Ответ `200 OK`

```json
{
  "articles": [ /* Article[] */ ],
  "has_more": true
}
```

`has_more: true` означает, что есть ещё страницы — передай `after=<последний id>` для следующей.

## GET /api/news/{id}

Полная статья по ID.

### Параметры пути

| Параметр | Тип   | Описание  |
|----------|-------|-----------|
| `id`     | int64 | ID статьи |

### Ответ `200 OK`

```json
{ /* Article */ }
```

### Ошибки

| Код | Описание             |
|-----|----------------------|
| 400 | Невалидный ID        |
| 404 | Статья не найдена    |

## GET /api/news/{id}/summary

Лёгкий ответ — только AI-поля без полного контента. Удобно для карточек в ленте.

### Ответ `200 OK`

```json
{
  "id": 42,
  "title": "Оригинальный заголовок",
  "ai_title": "AI-заголовок",
  "summary": "Краткое изложение от AI.",
  "tags": ["россия", "санкции", "экономика"]
}
```

Поля `ai_title`, `summary`, `tags` отсутствуют если AI-обогащение ещё не завершено.

## POST /api/news/{id}/rate

Оценить статью.

### Тело запроса

```json
{ "vote": "good" }
```

| Значение `vote` | Описание  |
|-----------------|-----------|
| `"good"`        | Лайк      |
| `"bad"`         | Дизлайк   |

### Ответ `200 OK`

```json
{
  "likes": 12,
  "dislikes": 3
}
```

### Ошибки

| Код | Описание                        |
|-----|---------------------------------|
| 400 | Невалидный ID или значение vote |
| 404 | Статья не найдена               |

## POST /api/news/{id}/save

Проанализировать статью с помощью Gemini и сохранить структурированную заметку в Obsidian.

Требует: `GEMINI_API_KEY` и `OBSIDIAN_VAULT_PATH` настроены на сервере.

### Тело запроса

Пустое или отсутствует.

### Ответ `200 OK`

```json
{
  "file": "новости/2026-03-05-nvidia-h100.md"
}
```

Путь возвращается относительно vault-директории.

### Ошибки

| Код | Описание                        |
|-----|---------------------------------|
| 400 | Невалидный ID                   |
| 404 | Статья не найдена               |
| 500 | Ошибка Gemini или записи файла  |
| 503 | Gemini не настроен на сервере   |

## Объект Article

```json
{
  "id": 42,
  "source_id": 7,
  "source": "Хабр",
  "title": "Go 1.23: что нового",
  "ai_title": "Go 1.23 добавляет range-over-func и улучшает toolchain",
  "summary": "В релизе Go 1.23 появилась возможность итерироваться по функциям...",
  "tags": ["golang", "программирование", "релиз"],
  "content": "Полный текст статьи...",
  "url": "https://habr.com/ru/articles/...",
  "image_url": "http://minio:9000/news-images/images/7/12345.jpg",
  "likes": 5,
  "dislikes": 0,
  "pub_date": "2026-03-04T12:00:00Z",
  "created_at": "2026-03-04T12:05:00Z"
}
```

### Описание полей

| Поле         | Тип            | Всегда | Описание |
|--------------|----------------|--------|----------|
| `id`         | int64          | да     | Уникальный ID статьи |
| `source_id`  | int64          | да     | ID источника |
| `source`     | string         | да     | Название источника |
| `title`      | string         | да     | Оригинальный заголовок |
| `ai_title`   | string         | нет    | AI-заголовок (пусто пока не обработано) |
| `summary`    | string         | нет    | AI-краткое изложение |
| `tags`       | string[]       | да     | AI-теги (пустой массив если нет) |
| `content`    | string         | да     | Полный текст |
| `url`        | string         | да     | Ссылка на оригинал |
| `image_url`  | string         | нет    | URL картинки в S3/MinIO |
| `likes`      | int            | да     | Число лайков |
| `dislikes`   | int            | да     | Число дизлайков |
| `pub_date`   | string (RFC3339) | нет  | Дата публикации из источника |
| `created_at` | string (RFC3339) | да   | Дата добавления в систему |

---

## Коды ошибок

| Код | Описание |
|-----|----------|
| 400 | Bad Request — невалидный параметр или тело |
| 404 | Not Found — статья не найдена |
| 500 | Internal Server Error — ошибка БД или внешнего сервиса |
| 503 | Service Unavailable — зависимый сервис не настроен |

Тело ошибки — plain text (кроме 503 от `/save`, там JSON с полем `error`).

## Примеры

### Получить последние 10 новостей

```http
GET /api/news?limit=10
```

### Прокрутка вниз (бесконечная лента)

```http
GET /api/news?limit=20&after=500
```

### Live-обновление (pull-to-refresh)

```http
GET /api/news?since=542
```

### Фильтрация по тегам

```http
GET /api/news?tags=россия,санкции&limit=20
```

### Комбинация: теги + пагинация

```http
GET /api/news?tags=golang&after=100&limit=10
```

### Оценить статью

```http
POST /api/news/42/rate
Content-Type: application/json

{"vote":"good"}
```

### Сохранить статью в Obsidian

```http
POST /api/news/42/save
```

```json
{"file": "новости/2026-03-05-go-123.md"}
```

### curl — лента с тегом

```bash
curl "http://localhost:8080/api/news?tags=golang&limit=5" | jq .
```

## Пагинация — схема работы

```
Первая загрузка:   GET /api/news?limit=20
                   → articles[0..19], has_more=true, последний id=480

Следующая страница: GET /api/news?limit=20&after=480
                   → articles[460..479], has_more=false

Live-обновление:   GET /api/news?since=499   (первый id из первой загрузки)
                   → новые статьи с id > 499
```
